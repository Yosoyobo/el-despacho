{% comment %} Tabla de datos TailAdmin canónica — Wave 3. Es el sucesor de _tabla.html para listas con orden + paginación. _tabla.html queda para tablas pasivas (detalles, resúmenes). Si tu vista es una lista con queryset, usa este. Params: - cabeceras: lista de dicts. Cada dict puede traer: - label (str, requerido) - sort_key (str, opcional): si está, la columna es clickeable y ordena por ese campo. Convención: el view recibe `?orden=foo` o `?orden=-foo`. - align ('left' | 'right' | 'center', default 'left') - clase_th (str, opcional): clases extras para el - filas_template (str, recomendado): path al template parcial que renderiza los ... del . Tu template recibe el contexto completo de la vista (clientes, proyectos, etc.). - filas_html (str, alternativa): bloque HTML pre-renderizado. Útil cuando ya tienes la cadena de filas. Se prefiere filas_template. - vacia_mensaje (str, opcional) - orden_actual (str): valor actual de ?orden= (ej. 'fecha' o '-fecha'). Usado para pintar la flecha + invertir el toggle al clic. - querystring_base (str): querystring SIN orden ni page (ej. 'q=foo&estado=activo'). Se concatena al link de orden para preservar filtros. - page_obj (opcional): si se pasa, renderiza la paginación TailAdmin al pie. Sticky thead: el partial agrega `sticky top-0` a — funciona dentro de `overflow-y-auto` (el wrapper aquí ya lo trae con `max-h-[70vh]`). Si quieres que NO scrollee, pasa `sin_scroll_vertical=True`. Acciones por fila: usa _tabla_acciones.html dentro de tus filas_html. {% endcomment %}
